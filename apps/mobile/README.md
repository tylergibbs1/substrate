# @substrate/mobile

Expo viewer target. **Post-v1 / viewer only** (PRD §3 non-goals, §9).

In v1 this is intentionally a stub: mobile authoring is a non-goal. The planned
shape is a read-only Expo app that connects to a running Substrate server over
the same WebSocket transport as the web editor and renders each slide's image
plus its slide prompt as the text-of-record. No prompt editing on mobile in v1.
